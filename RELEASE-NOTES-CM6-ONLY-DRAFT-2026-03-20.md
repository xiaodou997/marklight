# 发布说明草案（CM6 单引擎版）

## 版本概览
- 本版本完成编辑器内核收敛：正式下线 ProseMirror，统一使用 CodeMirror 6（CM6）。
- 目标是降低维护复杂度、减少双引擎分支带来的行为差异，并提升后续迭代效率。

## 主要变更
- 删除 ProseMirror 编辑器实现代码与相关插件目录。
- 移除 `prosemirror-*` 依赖及 `@types/prosemirror-tables`。
- 设置页不再提供编辑器引擎切换入口，固定为 CM6。
- 导出链路保持 Markdown 直出（微信/HTML），不再依赖 PM 文档树转换。

## 用户可感知变化
- 编辑体验统一为 CM6，不再支持在设置中切换回 PM。
- 历史配置中包含 `editorEngine` 字段的用户会自动兼容，不影响启动。
- 功能维持现有能力：实时渲染、搜索替换、快捷键、图片拖拽/粘贴、导出。

## 风险与已知事项
- 需重点关注 GUI 人工回归项：光标行为、显隐切换连续性、复杂文档导出样式。
- 构建已通过，但仍存在 chunk size 警告（不阻塞发布）。

## 升级说明
- 无需手工迁移数据文件。
- 如遇异常，请提交问题并附上：
- 系统版本与架构
- 复现步骤
- 样例 Markdown 文件（可脱敏）
- 控制台日志与错误截图

## 回滚策略
- 如线上出现阻塞问题，建议回滚到上一个稳定 tag/commit。
- 本版本后不再维护 PM 双引擎分支。
