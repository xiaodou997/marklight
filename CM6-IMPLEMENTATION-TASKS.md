# CM6 迁移执行清单

> 最后更新：2026-03-20

## 目标
- 在不影响现有 ProseMirror 稳定性的前提下，逐步落地 CodeMirror 6 编辑引擎。
- 采用双引擎并行策略，先可用、再增强、再默认切换。

## 风险追踪
- `Live Preview` 当前为正则方案，复杂嵌套语法仍需 AST 级增强。
- 导出主链路已改为 Markdown 直出（微信/HTML）；剩余风险集中在复杂文档样式的一致性验收。
- 部分高级语法（脚注、定义列表、缩写等）尚未完成 CM6 专项增强。
